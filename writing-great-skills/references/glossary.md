# Skill Writing Glossary

- `predictability`: the agent follows the intended process reliably across runs.
- `context load`: text that is always visible to the model because it is in metadata or loaded skill bodies.
- `cognitive load`: burden on the user to remember when and how to invoke a skill.
- `leading word`: a compact term that anchors behavior, such as "tight loop" or "tracer bullet".
- `progressive disclosure`: moving details from always-loaded text into linked files that are opened only when needed.
- `context pointer`: the sentence that tells the agent when to open a reference file.
- `completion criterion`: the observable condition that proves a step is done.
- `premature completion`: the agent stops because the task feels done before the completion criterion is actually met.
- `duplication`: the same behavioral instruction appears in multiple places.
- `no-op`: a sentence that does not change behavior compared with default agent behavior.
- `sediment`: stale text that remains because removing it feels riskier than leaving it.
- `sprawl`: a skill is too large even if its individual lines are useful.
